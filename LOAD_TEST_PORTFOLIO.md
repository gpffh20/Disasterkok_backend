## 1. 프로젝트 개요

### 1.1 서비스 특성

| 항목 | 내용 |
|------|------|
| 서비스명 | DisasterKok (재난 알림 커뮤니티) |
| 기술 스택 | Django 4.0 + DRF, PostgreSQL, Redis, WebSocket |
| 핵심 특징 | **재난 발생 시 30초 내 트래픽 10배 급증** |

### 1.2 트래픽 시나리오

```
트래픽
  ▲
  │                    ┌───┐
  │                   ╱     ╲   ← 전국 재난 (2,000명)
  │       ┌──┐       ╱       ╲
  │      ╱    ╲     ╱         ╲  ← 지역 재난 (500명)
  │─────╱──────╲───╱───────────╲─────
  │ 평상시 (100명)
  └──────────────────────────────────→ 시간
```

| 시나리오 | 동시 접속 | 상황 예시 |
|---------|----------|----------|
| 평상시 | 100명 | 일반적인 날 |
| 지역 재난 | 500명 | 특정 지역 지진/화재 |
| 전국 재난 | 2,000명+ | 장마/태풍 |

### 1.3 성능 목표 (SLO)

> **"2,000명 동시 접속 + 30초 내 급증에도 500ms 이내 응답, 0.5% 미만 에러율"**

| 지표 | 목표값 |
|------|--------|
| 평균 응답 시간 | < 300ms |
| 95% 응답 시간 | < 500ms |
| 에러율 | < 0.5% |
| CPU 사용률 | < 85% |
| 메모리 사용률 | < 90% |

### 1.4 테스트 대상 API

```
[인증]
POST /users/login/        - 로그인 (JWT 발급)

[게시글] - 주요 테스트 대상
GET  /posts/              - 목록 조회 (40%)  ← 가장 빈번
GET  /posts/{id}/         - 상세 조회 (25%)
POST /posts/{id}/likes/   - 좋아요 (15%)
POST /posts/              - 게시글 작성 (5%)

[알림]
GET  /notifications/              - 알림 목록 (10%)
GET  /notifications/unread-count/ - 미읽음 개수 (5%)
```

## 3. 테스트 환경 준비

### Step 1: JMeter 설치

```bash
# macOS
brew install jmeter

# 또는 직접 다운로드: https://jmeter.apache.org/download_jmeter.cgi

# JMeter 실행
jmeter
```

### Step 2: JMeter 플러그인 설치

```
JMeter 실행 → Options → Plugins Manager → Available Plugins

필수 설치:
☑ jpgc-perfmon (PerfMon Metrics Collector)  ← 서버 모니터링
☑ jpgc-casutg (Stepping Thread Group)       ← 단계별 부하
☑ jpgc-graphs-basic (기본 그래프)
```

### Step 3: PerfMon Server Agent 설치

> 서버(로컬/EC2)에서 CPU, Memory 등을 JMeter로 직접 수집

```bash
# 다운로드
wget https://github.com/undera/perfmon-agent/releases/download/2.2.3/ServerAgent-2.2.3.zip

# 압축 해제
unzip ServerAgent-2.2.3.zip

# 실행 (기본 포트: 4444)
cd ServerAgent-2.2.3
./startAgent.sh  # Linux/Mac
# 또는
startAgent.bat   # Windows
```

### Step 4: 테스트 데이터 생성

```python
# scripts/generate_test_data.py
import os, django
os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'config.settings.development')
django.setup()

from user.models import User
from post.models import Post

# 테스트 사용자 100명
for i in range(1, 101):
    User.objects.create_user(
        username=f'testuser{i}',
        email=f'test{i}@example.com',
        password='password123',
        nickname=f'테스터{i}'
    )

# 테스트 게시글 500개
users = User.objects.all()
for i in range(500):
    Post.objects.create(
        user=users[i % len(users)],
        title=f'테스트 게시글 {i}',
        content=f'테스트 내용 {i}'
    )

print(f"Created {User.objects.count()} users, {Post.objects.count()} posts")
```

```bash
python scripts/generate_test_data.py
```

## 4. JMeter 설정

### 4.1 Test Plan 구조

```
Test Plan
├── User Defined Variables
│   ├── BASE_URL = http://localhost:8000
│   └── SERVER_IP = localhost (PerfMon용)
│
├── Thread Group (Main Users)
│   ├── CSV Data Set Config (users.csv)
│   ├── HTTP Header Manager
│   │   └── Content-Type: application/json
│   │
│   ├── [로그인] HTTP Request
│   │   ├── POST ${BASE_URL}/users/login/
│   │   ├── JSON Extractor: access_token
│   │   └── Response Assertion: status=200
│   │
│   ├── Loop Controller
│   │   ├── HTTP Header Manager (Authorization)
│   │   ├── [게시글 목록] GET /posts/
│   │   ├── [게시글 상세] GET /posts/${post_id}/
│   │   ├── [좋아요] POST /posts/${post_id}/likes/
│   │   └── Uniform Random Timer: 1000-3000ms
│   │
│   └── [로그아웃] POST /users/logout/
│
├── jp@gc - PerfMon Metrics Collector    ← 서버 메트릭 수집
│   ├── CPU (${SERVER_IP}:4444)
│   └── Memory (${SERVER_IP}:4444)
│
└── Listeners
    ├── Summary Report
    ├── Aggregate Report
    ├── jp@gc - Response Times Over Time
    └── jp@gc - PerfMon Metrics Collector (그래프)
```

### 4.2 PerfMon Metrics Collector 설정

```
jp@gc - PerfMon Metrics Collector 추가 후:

┌────────────────────────────────────────────────────────┐
│ Host/IP       │ Port │ Metric to collect │ Metric     │
├───────────────┼──────┼───────────────────┼────────────┤
│ ${SERVER_IP}  │ 4444 │ CPU               │ combined   │
│ ${SERVER_IP}  │ 4444 │ Memory            │ usedperc   │
│ ${SERVER_IP}  │ 4444 │ Disks I/O         │ queue      │
│ ${SERVER_IP}  │ 4444 │ Network I/O       │ bytesrecv  │
└───────────────┴──────┴───────────────────┴────────────┘
```

### 4.3 JSON Extractor (로그인 토큰)

| 설정 | 값 |
|------|-----|
| Names of created variables | access_token |
| JSON Path expressions | $.access |
| Match No. | 1 |

### 4.4 Thread Group 설정 (테스트별)

| 테스트 | Threads | Ramp-up | Duration | 목적 |
|--------|---------|---------|----------|------|
| Baseline | 10 | 10초 | 5분 | 최소 부하에서 기본 성능 측정 |
| Stress | 100→2,000 (단계적) | 각 60초 | 각 5분 | 단계별 성능 + 한계점 파악 |
| Spike | 100→10,000 | 30초 | 10분 | 급격한 트래픽 급증 대응 |

```
┌─────────────────────────────────────────────────────────┐
│  테스트 유형별 목적                                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Baseline (10명, 5분)                                   │
│  └── "시스템이 정상 동작하는지 기본 확인"                 │
│                                                         │
│  Stress (100→500→1,000→2,000명, 단계별)                │
│  └── "각 단계에서 성능 측정 + 어디서 한계인지 파악"       │
│      (= 단계별 Load Test + 한계점 파악)                  │
│                                                         │
│  Spike (100→10,000명, 30초 내 급증)                     │
│  └── "전국 재난 상황 - 대규모 트래픽 폭증 시 대응"       │
│                                                         │
│  💡 10,000명 테스트는 Auto Scaling 또는                  │
│     다중 인스턴스 환경에서만 가능                        │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 7.6 적용 예시: 코드 (바이브코딩)

#### 문제 발견: N+1 쿼리

```
[발견 과정]
1. Load 테스트 중 GET /regions/ 응답시간 급증 확인
2. PerfMon에서 DB CPU 사용률 급증 확인
3. PostgreSQL 슬로우 쿼리 로그 확인
4. Django Debug Toolbar로 쿼리 수 확인 → 요청당 50개 쿼리!

[원인 분석]
- Region 조회 시 관련 객체를 매번 개별 조회
- 사용자당 지역 5개 × 10개 관련 쿼리 = 50개

[해결 (바이브코딩)]
"Region 조회 시 N+1 쿼리 발생. select_related/prefetch_related 적용해줘"
```

```python
# Before (N+1)
regions = Region.objects.filter(user=user)

# After (최적화)
regions = Region.objects.filter(user=user).select_related(
    'notification_setting'
).prefetch_related('tags')
```
